| File | Purpose |
|------|---------|
| **setup_board.py** | Creates your Trello board with all pipeline stages |
| **trello_ops.py** | CLI tool for all Trello operations |
| **trello_client.py** | Async Trello API client (used internally) |

### Agent Prompts (The AI Team)

Located in `agents/` directory — each is a Claude system prompt:

| File | Agent Role | Responsibility |
|------|------------|-----------------|
| **intake_analyst.md** | Intake Analyst | Structures raw ideas into Idea Briefs |
| **feature_decomposer.md** | Feature Decomposer | Breaks ideas into 3-7 concrete features |
| **spec_writer.md** | Spec Writer | Writes detailed feature specifications |
| **spec_reviewer.md** | Spec Reviewer | Reviews specs for quality & alignment |
| **prd_assembler.md** | PRD Assembler | Merges specs into a unified PRD |
| **final_reviewer.md** | Final Reviewer | Final proofread and polish |

### Generated Files (Created During Use)

These files are generated by the system and kept in `artifacts/`:

```
artifacts/
├── idea-{id}_intake.md              # Original idea brief
├── idea-{id}_spec_feature1.md       # Feature 1 spec
├── idea-{id}_spec_feature2.md       # Feature 2 spec
├── ... (one per feature)
└── idea-{id}_prd.md                 # Final PRD
```

These are **not included** in the distribution but are created when you use the system.

## Key Concepts

### The Pipeline

```
Raw Idea → Intake → Decompose → Spec Write → Spec Review → Assemble → Final Review → PRD
```

6 AI agents working together to transform ideas into professional product specs.

### The Trello Board

Your board IS your status dashboard. No separate logs or tracking. Watch cards move in real-time as agents process them.

### Claude Code as Orchestrator

You tell Claude Code (Cowork mode) to process the pipeline. Claude Code:
- Reads board state
- Spawns agents for each stage
- Attaches outputs as files
- Updates labels and moves cards
- Handles parallel processing
- Reports progress

### One Artifact Per Card

Each card carries exactly ONE file attachment (the output of that stage). When the next agent processes the card, it replaces the attachment with the new output. Comments are for lightweight notes (feedback, review decisions).

## Common Workflows

### Workflow 1: Simple Autonomous Run

```
1. python trello_ops.py submit "Your idea"
2. [In Claude Code] "Process the pipeline"
3. Wait 10-15 minutes
4. Download PRD from Trello Done column
```

### Workflow 2: Batch Processing

```
1. Submit 5 ideas
2. Process them all in parallel through Spec Write
3. Review intermediate specs
4. Let Claude Code continue through review & assembly
```

### Workflow 3: Manual Control

```
1. Process one stage at a time
2. Review outputs
3. Provide feedback via comments
4. Move to next stage
```

### Workflow 4: Customized Agents

```
1. Edit agents/spec_writer.md to change style
2. Submit idea
3. Let Claude Code process with your custom agent
```

## Support & Customization

### Editing Agent Prompts

Each agent in `agents/` is a Claude system prompt. You can:
- Change the output format
- Add or remove sections
- Change review criteria
- Adjust writing style

Edit the files and Claude Code will use your versions automatically.

### Adding Custom Stages

1. Add a new list to your Trello board
2. Update `board_config.json` with the new stage
3. Create a new agent prompt in `agents/`
4. Tell Claude Code to handle the new stage

### Integrating with Other Tools

Claude Code can:
- Export PRDs to your docs system
- Post summaries to Slack
- Create GitHub issues from specs
- Sync with your project management tool

## Code Quality & Safety

### Security

- No API keys are stored in the repo
- All credentials go in `.env` (git-ignored)
- All communication is via official APIs
- No data is logged or persisted (except your specs on Trello)

### Reliability

- Code is tested and production-ready
- All API errors are caught and reported
- Trello board is the source of truth (no separate database)
- Cards can be manually recovered if needed

### Customization Safety

- Editing agent prompts is safe (they're just text files)
- Editing `trello_ops.py` is safe (well-commented and modular)
- Breaking changes are unlikely with your local edits
