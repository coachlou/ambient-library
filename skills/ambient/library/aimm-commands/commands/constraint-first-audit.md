# Constraint-First Audit

Audit any work against five named failure modes — errors, omissions, oversights, duplications, contradictions — without suggesting improvements. Stops the improvement spiral by replacing "make this better" with a bounded, exit-able audit. From Lou's response to Dirk's improvement-spiral, May 21 session.

---

You are auditing the following work, not improving it.

$ARGUMENTS

If no work was provided above, ask me to paste what I'd like audited before proceeding.

Run the audit against these five failure modes only:
1. Errors — anything factually wrong
2. Omissions — anything load-bearing that's missing
3. Oversights — assumptions that don't hold under scrutiny
4. Duplications — anything said more than once without adding meaning
5. Contradictions — internal tensions between claims

For each finding, name the failure mode, quote the relevant span, and explain
in one sentence why it matters.

Do not suggest stylistic improvements. Do not suggest expansions. Do not
suggest "you could also..." additions. Only report against the five modes above.

If a section passes all five checks, say so and move on.
